# 权限管理设计文档

## 1. 设计目标

本项目采用轻量 RBAC 权限模型，为课程后台管理系统提供可演示、可维护、以前端体验为重点的权限体系。

当前版本采用固定角色权限，不再提供角色权限页面动态勾选功能。这样可以降低简历项目的复杂度，避免管理员误改权限后把自己锁死，同时保留菜单、路由、按钮和接口四层权限控制。

当前版本覆盖：

- 登录后返回用户角色与权限码
- 前端根据权限动态显示菜单
- 路由级权限守卫，无权限跳转 403 页面
- 按钮级权限控制，如新增、编辑、删除
- 后端接口级 403 校验，避免绕过前端直接调用接口
- 管理员可新增、删除教师/学生账号，并修改教师/学生角色
- admin 固定拥有全部权限，不允许被修改角色或删除

## 4. 角色权限分配

权限配置集中维护在：

```txt
server/src/permissions.js
client/src/permissions.ts
```

后端 `server/src/permissions.js` 是实际鉴权来源，前端 `client/src/permissions.ts` 是同名镜像，用于菜单、路由和按钮控制。后续新增权限时，需要保持两边权限码一致。

### admin

admin 使用 `Object.values(PERMISSIONS)` 自动拥有全部权限。

### teacher

```txt
dashboard:view
courses:view
courses:create
courses:update
students:view
students:create
students:update
summary:view
summary:create
summary:update
summary:delete
```

### student

```txt
dashboard:view
courses:view
students:view
summary:view
summary:create
summary:update
summary:delete
```

## 5. 前端实现结构

### 权限码配置

文件：

```txt
client/src/permissions.ts
```

职责：

- 统一维护前端权限码
- 导出 `PermissionCode` 类型
- 提供 `hasPermission` 和 `hasAnyPermission`
- 作为 UI 控制镜像，不作为最终安全来源

### 权限组件

文件：

```txt
client/src/components/Permission.tsx
```

典型用法：

```tsx
<Permission code={PERMISSIONS.COURSES_CREATE}>
  <Button>新增课程</Button>
</Permission>
```

用于声明式控制按钮、局部 UI 是否展示。

### 菜单配置

文件：

```txt
client/src/router/nav-config.tsx
```

每个菜单项绑定一个权限码，当前用户没有对应权限时，该菜单不会显示。

### 路由守卫

文件：

```txt
client/src/router/RequirePermission.tsx
```

进入页面前检查权限。没有权限时跳转到 `/403`。

### 路由懒加载

文件：

```txt
client/src/router/page-elements.tsx
```

页面组件使用 `React.lazy + Suspense` 按路由拆包加载。这样首次进入系统时不需要一次性加载所有页面、图表、Markdown 渲染和代码高亮相关代码。

## 6. 后端实现结构

### 固定权限字典

文件：

```txt
server/src/permissions.js
```

职责：

- 维护权限码
- 维护固定角色列表
- 维护 teacher/student 权限白名单
- `getPermissionsByRole(role)` 按角色返回固定权限

### 接口级鉴权

文件：

```txt
server/src/middleware/auth.js
```

`authenticateToken` 负责解析 token，`requirePermission(permission)` 负责检查当前用户是否拥有接口需要的权限。

### 账号管理

文件：

```txt
server/src/routes/system.js
```

当前支持：

- 查看账号
- 新增教师/学生账号
- 删除教师/学生账号
- 修改教师/学生角色
- 返回固定角色列表

当前不支持：

- 修改 admin 角色
- 删除 admin
- 动态修改角色权限

## 8. 维护约定

新增权限时按这个顺序改：

1. 先在 `server/src/permissions.js` 增加权限码和角色白名单。
2. 再在 `client/src/permissions.ts` 增加同名权限码。
3. 菜单、路由、按钮、接口分别接入对应权限。
4. 更新本文档的权限点和接口权限列表。
5. 执行 `npm run lint` 和 `npm run build` 验证前端。

当前版本选择固定权限，是为了让权限模型更稳定，减少动态配置带来的边界问题。对于简历项目来说，重点展示的是前端权限控制链路、路由守卫、按钮级权限和接口级鉴权，而不是复杂的权限配置后台。

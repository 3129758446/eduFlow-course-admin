# 权限管理设计文档

## 1. 设计目标

本项目采用轻量 RBAC 权限模型，目标是为课程后台管理系统提供一套可演示、可扩展、前端重点突出的权限体系。

当前版本重点覆盖：

- 登录后返回用户角色与权限码
- 前端根据权限动态显示菜单
- 路由级权限守卫，无权限跳转 403 页面
- 按钮级权限控制，如新增、编辑、删除
- 后端接口级 403 校验，避免绕过前端直接调用接口

当前版本已支持账号角色调整与角色权限配置，角色权限映射由数据库维护。

## 5. 前端实现结构

### 权限码配置

文件：

```txt
client/src/permissions.ts
```

职责：

- 统一维护权限码
- 导出 `PermissionCode` 类型
- 提供基础权限判断函数

这样页面中不直接散落字符串，减少拼写错误，也方便后续扩展。

### 权限组件

文件：

```txt
client/src/components/Permission.tsx
```

典型用法：

```tsx
<Permission code={PERMISSIONS.COURSES_CREATE}>
  <Button>新增课程</Button>
</Permission>
```

用于声明式控制按钮、局部 UI 是否展示。

### 菜单配置

文件：

```txt
client/src/router/nav-config.tsx
```

每个菜单项包含：

```ts
{
  key: "courses",
  path: "/courses",
  label: "课程管理",
  permission: PERMISSIONS.COURSES_VIEW,
}
```

`AppShell` 根据当前用户权限过滤菜单，用户没有权限的页面不会出现在侧边栏中。

### 路由守卫

文件：

```txt
client/src/router/RequirePermission.tsx
```

无权限访问时跳转到：

```txt
/403
```

并将缺失权限通过 `location.state.requiredPermission` 传给 403 页面。

### 403 页面

文件：

```txt
client/src/pages/ForbiddenPage.tsx
```

当前能力：

- 显示当前角色
- 显示缺失权限码
- 返回当前用户第一个可访问页面

### 页面按钮控制

课程页：

```txt
client/src/pages/CoursesPage.tsx
```

权限控制：

- `courses:create` 控制新增课程
- `courses:update` 控制编辑、发布/下架
- `courses:delete` 控制删除
- 没有编辑/删除权限时，隐藏整个操作列

账号管理页：

```txt
client/src/pages/AccountsPage.tsx
```

权限控制：

- `accounts:view` 控制账号管理菜单、路由和账号列表接口
- `accounts:updateRole` 控制角色选择与保存按钮

角色权限页：

```txt
client/src/pages/RolesPage.tsx
```

权限控制：

- `roles:view` 控制角色权限菜单、路由和角色权限接口
- `roles:updatePermissions` 控制权限勾选保存

学生页：

```txt
client/src/pages/StudentsPage.tsx
```

权限控制：

- `students:create` 控制新增学生
- `students:update` 控制编辑
- `students:delete` 控制删除
- 没有编辑/删除权限时，隐藏整个操作列

## 6. 后端实现结构

### 角色权限配置

文件：

```txt
server/src/permissions.js
```

职责：

- 维护默认 `PERMISSIONS`
- 维护默认角色与默认角色权限，用于数据库初始化和迁移
- 通过 `getPermissionsByRole(role)` 从数据库返回当前角色权限

### 数据库表

当前权限相关表：

```txt
roles
permissions
role_permissions
```

其中：

- `roles` 保存角色
- `permissions` 保存权限字典
- `role_permissions` 保存角色与权限的多对多关系

### 登录返回权限

文件：

```txt
server/src/routes/auth.js
```

登录和 `/auth/me` 都会返回：

```ts
permissions: string[]
```

前端刷新页面后仍可恢复权限状态。

### 接口权限校验

文件：

```txt
server/src/middleware/auth.js
```

新增：

```js
requirePermission(permission)
```

接口使用示例：

```js
router.post(
  '/',
  authenticateToken,
  requirePermission(PERMISSIONS.COURSES_CREATE),
  async (ctx) => {}
);
```

这样即使用户绕过前端按钮直接调用接口，也会返回 403。

## 7. 当前接口权限规则

### 工作台

```txt
GET /api/dashboard -> dashboard:view
```

### 课程

```txt
GET    /api/courses              -> courses:view
GET    /api/courses/categories   -> courses:view
GET    /api/courses/:id          -> courses:view
POST   /api/courses              -> courses:create
PUT    /api/courses/:id          -> courses:update
DELETE /api/courses/:id          -> courses:delete
PATCH  /api/courses/:id/status   -> courses:update
```

### 学生

```txt
GET    /api/students             -> students:view
GET    /api/students/classes     -> students:view
GET    /api/students/check-no    -> students:update
GET    /api/students/:id         -> students:view
POST   /api/students             -> students:create
PUT    /api/students/:id         -> students:update
DELETE /api/students/:id         -> students:delete
```

### 学习总结

```txt
GET /api/summary -> summary:view
```

### 系统管理

```txt
GET   /api/system/users                    -> accounts:view
PATCH /api/system/users/:id/role           -> accounts:updateRole
GET   /api/system/roles                    -> roles:view
PUT   /api/system/roles/:code/permissions  -> roles:updatePermissions
```

## 8. 后续扩展建议

### 阶段一：账号管理增强

当前已支持查看账号和修改账号角色。后续可继续补充：

- 新增账号
- 重置密码
- 禁用/启用账号
- 防止管理员误改自己的关键权限

### 阶段二：角色权限管理增强

当前已支持查看角色、勾选权限并保存。后续可继续补充：

- 新增角色
- 删除角色
- 权限变更二次确认
- 保存后展示影响账号数量

### 阶段三：权限字典下发增强

后端提供权限字典接口：

```txt
GET /api/permissions
```

前端可以用它渲染角色授权页面，避免前端手写权限选项。

### 阶段四：权限测试

建议补充测试：

- student 不显示新增、编辑、删除按钮
- teacher 不显示删除按钮
- admin 显示全部操作
- 无权限访问路由跳转 403
- 无权限调用接口返回 403
